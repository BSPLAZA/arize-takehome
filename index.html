Existing Phoenix and AX evals catch hallucination and wrong-tool-selection. They don't catch an agent that asserts an action without firing the tool. I named the class tool bypass, wrote an eval matching Phoenix's existing module patterns, and ran it against 169 real production turns: 7 real phantoms, 4 false positives.

The runtime story is already shipped on AX. The eval registers as a continuous online task, scores every Anthropic span automatically, and surfaces the verdict in the trace UI. The 30-day commit is about productizing the surface for AX customers, not about whether the eval works.
